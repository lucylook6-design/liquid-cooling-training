## 🎯 方案A：自动化同步（3步完成）

### 第1步：创建飞书应用（5分钟）

**操作步骤：**

1. **访问飞书开放平台**
   ```
   https://open.feishu.cn/
   ```

2. **创建应用**
   - 点击【控制台】→【创建企业自建应用】
   - 应用名称：`液冷培训同步工具`
   - 应用描述：`用于同步培训内容到飞书云文档`
   - 点击【创建】

3. **获取凭证**
   - 在应用详情页，找到【凭证与基础信息】
   - 复制保存以下信息：
     * **App ID**（形如：cli_xxxxxx）
     * **App Secret**（点击查看并复制）

4. **添加权限**
   - 点击【权限管理】
   - 搜索并添加以下权限：
     * `docx:document` - 文档权限
     * `drive:drive` - 云空间权限
   - 点击【发布版本】使权限生效

5. **配置重定向URL**
   - 点击【安全设置】
   - 在【重定向URL】中添加：
     ```
     http://localhost:3000/callback
     ```
   - 点击【保存】

✅ **完成标志：** 您已获得App ID和App Secret

## 🔧 故障排查

### 问题1：登录失败

**症状：** 浏览器授权后，终端显示"登录失败"

**解决方案：**
1. 检查App ID和App Secret是否正确
2. 确认应用权限已添加并发布
3. 确认重定向URL配置为 `http://localhost:3000/callback`
4. 重新运行 `./setup-feishu-mcp.sh`

### 问题2：MCP工具不可用

**症状：** Kiro中无法使用飞书MCP工具

**解决方案：**
1. 确认已重启Kiro
2. 检查 `~/.kiro/settings/mcp.json` 配置是否正确
3. 在终端运行测试：
   ```bash
   npx -y @larksuiteoapi/lark-mcp mcp -a <your_app_id> -s <your_app_secret> --oauth --token-mode user_access_token
   ```

### 问题3：无法创建文档

**症状：** AI提示"权限不足"或"无法创建文档"

**解决方案：**
1. 确认应用权限包含：
   - `docx:document`
   - `drive:drive`
2. 确认权限已发布（点击【发布版本】）
3. 重新登录：
   ```bash
   npx -y @larksuiteoapi/lark-mcp login -a <your_app_id> -s <your_app_secret> --scope "offline_access docx:document drive:drive"
   ```

### 问题4：内容格式错乱

**症状：** 飞书文档中的表格、列表格式不正确

**解决方案：**
- Markdown转换有限制，建议：
  1. 在飞书中手动调整格式
  2. 或者直接分享培训网站链接（推荐）
